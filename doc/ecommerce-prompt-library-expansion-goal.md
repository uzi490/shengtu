# 商图工坊电商提示词库扩展目标

## 目标

把商图工坊的“电商提示词库”从通用模板扩展为覆盖多行业、多平台、多场景的电商作图工作流库。模板必须来源清楚、被市场认可、可改写为中文商家可用提示词，并能一键生成画布节点工作流。

## 认可标准

提示词来源优先按 GitHub 公开认可度排序，不直接照搬原文，必须改写为商图工坊自己的中文电商工作流模板。

优先级：

1. GitHub stars 高、持续更新、包含预览图或结构化 prompt 的仓库。
2. 明确适配 GPT Image 2、Nano Banana、Midjourney、Stable Diffusion、Flux 等图像模型的 prompt 集。
3. 能落到电商真实场景：商品主图、详情页、模特图、场景图、广告图、海报、包装、珠宝、美妆、服装、家居、数码、食品等。
4. license 清楚或允许参考学习；如果 license 不清楚，只做风格和结构参考，不复制原文。
5. 每个模板必须有“保留商品真实细节”“不生成乱码文字”“不虚构功能/品牌/价格”等安全约束。

## 行业分类目标

第一阶段先补这些行业：

1. 服装鞋包：模特换装、隐形模特、街拍、平铺搭配、鞋类细节、包袋质感。
2. 美妆护肤：瓶身主图、质地特写、浴室/梳妆台场景、成分感、礼盒套装。
3. 食品饮料：餐桌场景、包装主图、食欲氛围、节日礼盒、冷饮/热饮质感。
4. 家居家纺：客厅/卧室场景、材质纹理、尺寸感、套装搭配。
5. 数码家电：科技主图、使用场景、接口细节、结构卖点、桌面办公场景。
6. 母婴玩具：安全温暖场景、家庭使用、材质细节、礼盒彩色场景。
7. 珠宝饰品：微距质感、模特佩戴、礼盒场景、金属/宝石光泽。
8. 运动户外：动感场景、装备搭配、防水耐用细节、户外光影。
9. 宠物用品：宠物使用场景、家居融合、套装平铺、温暖生活方式。
10. 节日营销：618、双 11、年货节、情人节、母亲节、圣诞、新品上市。

## 工作流类型目标

每个行业尽量覆盖下面 6 类：

1. 主图：白底主图、高级主图、透明/浅色背景、平台规范主图。
2. 场景图：真实使用场景、生活方式、礼盒送礼、套装搭配。
3. 图生图编辑：换背景、换模特、换场景、保留商品主体。
4. 详情页：局部细节、材质纹理、结构卖点、尺寸对比。
5. 营销海报：节日促销、大促活动、信息流广告、短视频封面。
6. 爆款复刻：参考高转化构图，不复制品牌和版权元素。

## 模板写入标准

每个模板写入 `src/config/ecommercePrompts.js` 时必须包含：

- `id`：英文 kebab-case，稳定可追踪。
- `category`：沿用 `main-image`、`scene`、`detail`、`poster`、`background`、`style`。
- `name`：中文短名，商家能看懂。
- `description`：一句话说明用途。
- `size`：默认按场景选择 `2048x2048`、`1440x2560` 或 `2560x1440`。
- `needsProductImage: true`：默认要求上传商品参考图。
- `prompt`：必须包含保留商品、场景要求、构图光线、限制项。

## 执行批次

### 批次 1：来源采集和筛选

- 读取第一批 GitHub 来源池 README / prompt 文件。
- 记录 stars、license、更新时间、可参考方向。
- 标记哪些可直接参考结构，哪些只可学习风格。
- 输出 `doc/ecommerce-prompt-source-audit.md`。

### 批次 2：行业模板扩展

- 每个行业先写 3 个模板，优先覆盖主图、场景图、详情页。
- 第一轮目标新增 30 个模板。
- 保持模板总数可控，不为了数量牺牲质量。

### 批次 3：工作流体验优化

- 电商提示词库增加行业筛选或二级标签。
- 首页推荐优先展示高频电商场景。
- 每个卡片显示“适合行业 / 需要参考图 / 推荐尺寸”。

### 批次 4：测试和上线

- 本地 `npm run build`。
- 脚本检查模板数量、id 唯一、分类合法、必填字段完整。
- 部署首尔测试。
- 用户真实上传商品图抽测 10 个模板。
- 通过后合并 `main`。

## 验收标准

- 至少 50 个高质量电商模板。
- 至少覆盖 10 个行业。
- 每个模板都有明确电商用途和参考图保留约束。
- 用户不用写复杂提示词，点模板后只需要填产品信息、上传参考图即可生成工作流。
- 不直接复制不明 license 的原始 prompt；所有模板改写成商图工坊自有中文模板。
